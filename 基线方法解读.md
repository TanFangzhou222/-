# 基线方法解读

更新日期：2026-04-16

## 1. 当前已完成的基线范围
当前第一阶段分解实验已经实现并跑通 7 个基线：

1. `total_field_highpass`
2. `wavelet_denoise`
3. `lowrank_sparse`
4. `emd`
5. `vmd`
6. `obf`
7. `gradient_obf`

其中：

- 前 6 个方法主要面向总场单通道 `signal`
- `gradient_obf` 面向梯度单通道 `signal_1`

因此在解读结果时，要注意：`gradient_obf` 与其它总场方法不是完全同输入对比，它更适合被看作“梯度路线代表”。

## 2. lowrank_sparse 的原理、流程与输出
### 2.1 方法目标
`lowrank_sparse` 的目标不是简单滤波，而是把总场单通道信号拆成：

- 一个低秩背景部分
- 一个稀疏异常部分

它对应的是 README 里“低秩稀疏分解基线”这一路线。

### 2.2 核心思想
原始一维信号 `x(t)` 会先被重排成 Hankel 轨迹矩阵：

```math
H(x) \in \mathbb{R}^{L \times K}
```

其中：

- `L` 是滑动窗口长度，当前默认 `hankel_window = 96`
- `K = N - L + 1`

这样做的理由是：如果一段信号主要由缓慢变化背景构成，那么它的轨迹矩阵通常更接近低秩；而局部异常或尖峰更容易表现为稀疏扰动。

### 2.3 低秩稀疏分解
当前实现中，对 Hankel 矩阵做近似的鲁棒 PCA / PCP 分解：

```math
H(x) = L + S
```

其中：

- `L` 是低秩矩阵，对应背景结构
- `S` 是稀疏矩阵，对应局部异常或突变结构

代码中采用的是“奇异值软阈值 + 元素级软阈值 + 对偶更新”的迭代方式：

- 对 `L` 做奇异值阈值收缩，保留主要低秩结构
- 对 `S` 做元素级软阈值，保留稀疏异常项
- 通过残差和对偶变量逐步逼近分解结果

当前默认参数：

- `hankel_window: 96`
- `lambda_scale: 1.0`
- `max_iter: 25`
- `tol: 1.0e-5`
- `max_rank: 12`

### 2.4 反 Hankel 重建
分解得到矩阵 `L` 和 `S` 后，代码通过对角平均把它们重新投影回一维序列：

- `lowrank_component`
- `sparse_component`

当前定义中：

- `background = lowrank_component`
- `residual = signal - background`

因此，低秩部分被解释为背景，剩余部分被解释为异常残差。

### 2.5 统一后处理
`lowrank_sparse` 并不会直接把矩阵分解结果当最终响应输出，而是继续走统一单通道后处理：

1. 对 `residual` 做短窗平滑
2. 用局部 MAD 做尺度归一化
3. 取绝对值形成 `enhanced_response`

因此最终响应可近似写成：

```math
response(t) = \left|\frac{\operatorname{Mean}_{9}(r)(t)}{\operatorname{MAD}_{101}(r_s)(t)}\right|
```

### 2.6 当前输出
`lowrank_sparse` 当前在逐点结果表中会额外输出：

- `lowrank_component`
- `sparse_component`
- `background_estimate`
- `residual`
- `enhanced_response`

同时还会记录：

- `pcp_iterations`
- `pcp_estimated_rank`
- `pcp_final_rel_error`
- `reconstruction_error`

### 2.7 当前运行表现
按当前全量 29 文件结果，`lowrank_sparse` 的平均行为是：

- 平均迭代次数：`20.72`
- 中位迭代次数：`19`
- 平均估计秩：`2.97`
- 中位估计秩：`2`
- 平均重建误差：`7.91e-06`
- 最大重建误差：`1.69e-05`

说明：

- 当前实现数值上是稳定收敛的
- 低秩背景在大多数轨迹上只需要很低的有效秩
- 分解后的重建误差很小，说明 `L + S` 能较好解释原始序列

## 3. 七个基线的结果差异解读
### 3.1 总体量化结果
按当前 `comparison_overall.csv`，7 个方法的总体表现如下：

| 方法 | 输入 | mean_residual_rms | mean_response_mean | max_peak_response |
|---|---|---:|---:|---:|
| total_field_highpass | signal | 3.2922 | 5.1445 | 9271.8360 |
| wavelet_denoise | signal | 2.0578 | 0.8850 | 198.5455 |
| lowrank_sparse | signal | 1.8969 | 0.9677 | 348.8962 |
| emd | signal | 0.7808 | 1.0373 | 281.9112 |
| vmd | signal | 0.0283 | 0.8056 | 29.5244 |
| obf | signal | 5.6065 | 3.2845 | 290.8078 |
| gradient_obf | signal_1 | 0.9830 | 2.2124 | 1049.6650 |

### 3.2 结果分层理解
当前这 7 个方法大致可以分成 4 类风格：

#### A. 激进增强型
- `total_field_highpass`

它的特点是：

- 峰值响应远高于其它方法
- 局部异常会被强烈放大
- 同时对局部噪声和尖峰也更敏感

从分轨迹统计看：

- 在 29 条轨迹里，有 23 条的 `peak_response` 是它最大
- `mean_response_mean` 也是全组最高
- 但 `mean_response_std` 和 `max_peak_response` 都非常大，说明稳定性较差

结论：

- 适合做“最强放大基线”
- 不适合直接作为稳健结构分离方法代表

#### B. 保守净化型
- `wavelet_denoise`
- `vmd`

`wavelet_denoise`：

- 比高通平滑得多
- 背景抑制相对稳健
- 但异常峰值明显被压低

`vmd`：

- 当前参数下 `residual_rms` 最小
- 29 条轨迹全部是最小 `residual_rms` 获胜者
- 说明它当前最偏向“把一切都解释成可分解背景/模态”

结论：

- `wavelet_denoise` 更像“稳健去噪器”
- `vmd` 当前更像“净化过度”的分解器，需要调参后再判断

#### C. 结构分离平衡型
- `emd`
- `lowrank_sparse`

这是目前总场单通道里最值得关注的一组。

`emd`：

- `mean_residual_rms` 不高
- `mean_peak_response` 比小波明显更强
- 说明它留下来的总体能量不大，但还能保住局部结构峰值

`lowrank_sparse`：

- `mean_residual_rms = 1.8969`，比高通和 OBF 明显低，比 EMD 略高
- `mean_peak_response = 42.10`，接近 OBF 和 EMD 的量级
- `max_peak_response = 348.90`，强于小波、EMD、OBF、VMD

更重要的是：

- `lowrank_sparse` 在 `A1-18.1` 和 `A1-32.1` 两条轨迹上拿到了最高 `peak_response`
- 说明它在某些“局部异常嵌入复杂背景”的轨迹上，确实比普通滤波和模态方法更擅长把结构性异常拉出来

结论：

- `emd` 更偏局部模态保留
- `lowrank_sparse` 更偏背景结构与异常结构分离
- 这两者是当前总场单通道里最值得继续保留和细化的一组

#### D. 背景拟合型
- `obf`
- `gradient_obf`

`obf`：

- `mean_residual_rms` 是总场方法里最高
- 但峰值响应没有高通过激
- 它更像“低阶背景拟合后剩余宽残差”的方法

分轨迹看：

- 在 29 条轨迹里，有 11 条是 `response_mean` 最高
- 这说明它常常留下更宽、更持续的残差，而不是尖锐峰值

`gradient_obf`：

- 输入是梯度单通道，不和总场方法完全同口径
- `mean_peak_response = 157.38`，明显高于大多数总场方法
- `max_peak_response = 1049.67`，说明梯度路线对局部异常更敏感

结论：

- `obf` 更适合作为“背景拟合对照组”
- `gradient_obf` 是当前梯度单通道路线里值得保留的代表方法

## 4. 当前 7 个基线的综合判断
### 4.1 如果关注“异常峰值尽量被抬起来”
优先级大致是：

- `total_field_highpass`
- `gradient_obf`
- `lowrank_sparse`

但要注意：

- `total_field_highpass` 代价是波动极大
- `gradient_obf` 不是总场输入

### 4.2 如果关注“结构分离的平衡性”
优先级大致是：

- `lowrank_sparse`
- `emd`
- `wavelet_denoise`

其中：

- `lowrank_sparse` 更偏结构背景分离
- `emd` 更偏局部模态保留
- `wavelet_denoise` 更偏保守平滑

### 4.3 如果关注“尽量把背景压到最干净”
当前 `vmd` 最强，但这不是完全意义上的优点，因为它也很可能把异常一起洗掉了。

### 4.4 如果关注“梯度路线是否值得继续保留”
答案是值得。当前 `gradient_obf` 的峰值表现说明：

- 梯度单通道对局部异常是敏感的
- 后续梯度路线不应删掉

## 5. 当前最有价值的结论
基于目前结果，可以得出几条对下一步开发最有用的判断：

1. `total_field_highpass` 可以继续保留，但更适合作为“激进增强对照组”，不适合作为主结构分离基线。
2. `vmd` 当前参数明显偏向净化过度，后续应优先调参，而不是直接拿当前结果下结论。
3. `emd` 在总场单通道里依然是一个较平衡的传统分解基线。
4. `lowrank_sparse` 已经具备独立保留价值。它不是最激进的方法，但在复杂背景轨迹上能表现出结构分离优势。
5. `obf` 适合作为背景拟合对照组继续保留。
6. `gradient_obf` 说明梯度单通道路线有效，应在后续梯度分解实验中继续使用。

## 6. 下一步建议
基于当前结果，后续优先级建议如下：

1. 先对 `vmd` 做参数重调，重点检查 `num_modes`、`keep_modes`、`alpha`。
2. 对 `lowrank_sparse` 做参数扫描，重点检查：
   - `hankel_window`
   - `lambda_scale`
   - `max_rank`
3. 在后续论文或报告里，将总场单通道主对照优先保留：
   - `highpass`
   - `wavelet_denoise`
   - `emd`
   - `lowrank_sparse`
   - `obf`
4. 梯度路线单独保留 `gradient_obf` 作为代表基线。
