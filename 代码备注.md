# SimpleDetector 代码备注

## 1. 文件定位
当前检测逻辑主要位于：

- `src/detect/simple_detector.py`
- `configs/experiment/simple_detector.yaml`

本文件说明的是**当前代码版本**下 `SimpleDetector` 的检测原理，而不是最早的简化版逻辑。

## 2. 当前任务中的作用
`SimpleDetector` 是一个最小可运行的前端候选检测器，用于把表示层输出的连续序列进一步转成“候选异常片段”。

它不是学习模型，而是一个基于规则和统计量的检测器。整体目标是：

1. 去掉慢变背景
2. 把局部异常按局部噪声尺度归一化
3. 把高分点转成连续候选片段

## 3. 当前默认输入
按照当前配置 `configs/experiment/simple_detector.yaml`，默认运行对象为：

- 表示类型：`single_channel`
- 输入列：`signal`

因此，当前版本实际上是在单通道信号上做异常候选检测。

## 4. 检测主流程
### 4.1 读取信号
从输入表中读取 `signal_columns` 指定的列。当前默认只有：

- `signal`

如果以后切换到 `real_concat` 或 `dual_mode`，也可以通过配置改成多列联合打分。

### 4.2 局部去背景
如果 `background_window > 1`，代码会先做局部背景估计，再从原信号中减掉背景。

当前配置为：

- `background_window: 301`
- `background_method: median`

因此当前等价于：

```math
r(t) = x(t) - \operatorname{Median}_{301}(x)(t)
```

其中：

- `x(t)` 为原始输入信号
- `r(t)` 为去背景后的局部残差

这里采用滑动中值而不是滑动均值，是为了降低尖峰和局部异常对背景估计的污染。

### 4.3 局部尺度归一化
如果 `normalize_columns: true`，代码会把残差再除以局部尺度估计，从而得到相对异常强度。

当前配置为：

- `scale_window: 101`
- `scale_method: mad`
- `scale_floor_ratio: 0.25`
- `scale_epsilon: 1.0e-6`

因此当前使用的是滑动 MAD 尺度：

```math
u(t) = \frac{r(t)}{\operatorname{MAD}_{101}(r)(t)}
```

为了避免局部尺度过小导致分数异常放大，代码对尺度设置了下限，使用全局尺度的一部分作为 floor。

这一步的意义是：

- 在噪声较大的区段不过度敏感
- 在噪声较小的区段不过度宽松
- 让检测分数更接近“相对异常程度”而不是原始幅值

### 4.4 绝对值与平滑
当前配置中：

- `use_absolute: true`
- `smooth_window: 9`

所以代码会先取绝对值，再做 9 点滑动均值：

```math
score(t) = \operatorname{Mean}_{9}(|u(t)|)
```

含义是：

- 不区分正异常和负异常，只关心偏离强度
- 抑制单点毛刺，让候选更连续

### 4.5 边缘保护
当前配置中：

```yaml
edge_guard:
  enabled: true
  margin: null
```

当 `margin` 为空时，代码会自动根据背景窗、尺度窗和平滑窗估算边缘保护宽度，并把前后若干点的检测分数直接压成 0。

其目的是避免滑动窗口在边界处因样本不完整而产生假高分。

## 5. 阈值判决逻辑
阈值计算支持 4 种方式：

- `fixed`
- `quantile`
- `zscore`
- `mad`

当前配置为：

```yaml
threshold:
  method: fixed
  value: 8.0
```

因此当前的判决规则非常直接：

```math
score(t) \ge 8.0 \Rightarrow \text{异常候选点}
```

这说明当前版本采用的是**固定门限检测**，而不是按当前样本分布自适应估阈值。

## 6. 候选片段生成逻辑
代码不会把超阈值点直接当最终结果，而是进一步构造成连续候选片段。

### 6.1 连续超阈值点形成原始片段
先根据：

```math
score(t) \ge threshold
```

得到布尔掩码，然后把连续为真的区间提取出来。

### 6.2 相邻片段合并
当前配置：

- `gap_tolerance: 3`

如果两个候选片段之间的间隔不超过 3 个点，则把它们合并成一个更长的片段。

这样可以避免一个真实异常因局部短暂跌破阈值而被切碎。

### 6.3 过滤过短片段
当前配置：

- `min_length: 7`

合并后，如果某个片段长度小于 7 个采样点，就丢弃。

这样可以去掉很多短毛刺型误报。

## 7. 当前版本的近似公式总结
按当前默认配置，单通道检测过程可以近似写成：

```math
x(t) = signal
```

```math
r(t) = x(t) - \operatorname{Median}_{301}(x)(t)
```

```math
a(t) = \operatorname{MAD}_{101}(r)(t)
```

```math
u(t) = \frac{r(t)}{a(t)}
```

```math
score(t) = \operatorname{Mean}_{9}(|u(t)|)
```

然后：

```math
score(t) \ge 8.0
```

得到候选点，再经过：

- 短间隔合并
- 最短长度约束

最终得到候选异常片段。

## 8. 输出结果
`SimpleDetector` 主要输出两类结果。

### 8.1 annotated_frame
在原始输入表基础上增加：

- `detection_score`
- `is_candidate`

其中：

- `detection_score` 是每个采样点的最终检测分数
- `is_candidate` 表示该点是否超过阈值

### 8.2 candidates
每个候选片段会输出：

- `candidate_id`
- `start_index`
- `end_index`
- `num_points`
- `axis_start`
- `axis_end`
- `peak_sample_index`
- `peak_axis`
- `peak_score`

这使得后续可以继续做：

- 候选事件构造
- 边界精化
- 表示方法对比实验

## 9. 这套逻辑的本质
当前版本的 `SimpleDetector`，本质上不是简单的“幅值阈值器”，而是一个：

**局部去背景 + 局部稳健归一化 + 固定阈值 + 连续片段合并** 的候选检测器。

它更适合第一阶段当前的目标：

- 先快速把表示层结果转成可分析的候选片段
- 建立最小闭环
- 为后续表示有效性实验和候选构造实验提供基础输出

## 10. 当前局限
这套检测器仍然是最小基线，当前还没有：

- 多特征判别学习
- 自适应事件边界精化
- 候选质量评分
- 事件级语义编码
- 四元数域下的结构检测头

因此，它更适合做第一阶段前期对比和流程打通，不适合作为最终检测模型。

## 11. 总场单通道传统前端基线
当前新增的传统前端基线主要位于：

- `src/methods/filters.py`
- `src/experiments/exp_decomposition.py`
- `configs/experiment/exp_decomposition.yaml`

这一部分不是最终检测器，而是一个**总场单通道前端增强模块**。它的任务是先把慢变背景压掉，再把局部异常响应凸显出来，为后续检测或方法对比提供统一前端输出。

## 12. 当前默认输入
按照当前配置 `configs/experiment/exp_decomposition.yaml`，默认输入为：

- 输入目录：`data/processed/single_channel`
- 输入列：`signal`
- 表示类型：总场单通道

当前单个输入 CSV 里主要有：

- `sample_index`
- `axis`
- `baseline_distance`
- `signal`

真正参与计算的是 `signal`，其余列主要用于保留采样顺序和轨迹元信息。

## 13. 前端处理主流程
### 13.1 局部背景估计
当前配置为：

- `background_window: 301`
- `background_method: median`

因此代码会先用 301 点滑动中值估计背景：

```math
b(t) = \operatorname{Median}_{301}(x)(t)
```

其中 `x(t)` 是总场输入信号，`b(t)` 是局部背景估计。

### 13.2 高通残差提取
随后从原信号中减去背景，得到残差：

```math
r(t) = x(t) - b(t)
```

这一步就是当前代码中的“高通”核心。慢变趋势和大尺度背景会被压掉，局部异常和突变更容易保留下来。

### 13.3 残差平滑
当前配置：

- `residual_smooth_window: 9`

因此代码会对残差再做一个 9 点滑动均值平滑，得到 `residual_smoothed`。这一步主要用于抑制尖锐毛刺，让后续增强响应更稳定。

### 13.4 局部尺度归一化
当前配置为：

- `scale_window: 101`
- `scale_method: mad`
- `scale_floor_ratio: 0.25`
- `scale_epsilon: 1.0e-6`

因此代码会对平滑残差使用 101 点滑动 MAD 做局部尺度估计：

```math
a(t) = \operatorname{MAD}_{101}(r_s)(t)
```

再形成归一化残差：

```math
u(t) = \frac{r_s(t)}{a(t)}
```

其中 `r_s(t)` 表示平滑后的残差。为了避免局部尺度过小导致分数被异常放大，代码还设置了基于全局尺度的下限保护。

### 13.5 增强响应构造
当前配置中：

- `absolute_response: true`

因此最终增强响应为：

```math
response(t) = |u(t)|
```

这说明当前前端只关心“偏离背景有多强”，不区分正异常和负异常。

## 16. 这套前端的本质
这套代码本质上是一个：

**滑动中值去背景 + 高通残差提取 + 局部 MAD 稳健归一化** 的传统单通道前端基线。

它的作用不是直接给出最终目标事件，而是先把总场中的慢变背景与局部异常分开，让后续检测器或结构化分解方法有一个更干净的输入。

## 17. 当前局限
这套前端目前仍然是最小基线，当前还没有：

- 梯度单通道分支
- 小波、EMD、VMD 等分解方法
- 多通道联合前端
- 自适应事件级评分
- 与四元数统一表示的结构耦合前端

因此它更适合作为第一阶段“传统单通道基线”的起点，而不是最终方法。
