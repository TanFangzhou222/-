# 面向 TIM 的异常目标检测新实验框架

更新时间：2026-04-18

## 1. 目标重定义

当前仓库的主线更偏向“信号分解后观察异常残差”，这条路线适合做前端方法对比，但不够适合作为 `IEEE Transactions on Instrumentation and Measurement (TIM)` 级别论文的最终目标。原因主要有三点：

1. 任务定义偏弱，当前更多是在做“分解是否有效”，而不是“目标是否被可靠检测、定位和识别”。
2. 物理语义偏弱，分解出的分量并不天然对应船锚、潜艇等具体目标。
3. 评价闭环偏弱，目前更接近前端增强对比，缺少目标级指标、泛化验证和测量链不确定性分析。

因此建议将项目最终目标重构为：

**基于真实双传感器磁测背景和物理仿真目标注入，研究弱异常目标的增强、匹配、检测与定位方法。**

更具体地，新的研究任务应分成三个层次：

1. **事件检测**：在真实背景轨迹中判断是否存在目标异常事件。
2. **目标定位**：给出异常中心位置或区间边界。
3. **目标识别/检索**：判断最可能的目标类型或模板族，例如船锚、锚钩、潜艇、未知铁磁体。

## 2. 建议的论文主线

建议论文主线不要再写成“某种分解方法优于另一种分解方法”，而改成：

**Physics-guided simulation injection + multi-view enhancement + template matching + event-level validation**

对应的核心科学问题可以写成：

1. 在真实海洋磁背景样本稀缺且目标标签缺失的情况下，如何构造可信的半实物目标检测基准？
2. 在双传感器磁测条件下，如何利用总场、梯度场及其耦合表征增强弱目标响应？
3. 如何将物理仿真模板与真实测量背景结合，实现低虚警的异常目标匹配检测？

## 3. 适合 TIM 的创新点

如果要冲击 TIM，建议把创新点聚焦在“测量方法 + 物理一致性 + 可复现实验体系”，而不是只讲算法。

可以围绕以下 3 个贡献组织全文：

1. **半实物基准构建**：提出一种“真实背景 + 仿真目标注入”的磁异常检测数据生成框架，在缺少真实标注目标的情况下建立可控、可复现、可定量评价的测试基准。
2. **多视图增强与匹配**：提出一种面向双传感器磁测的多视图增强和模板匹配方法，同时利用总场、梯度场、双模态耦合响应和几何一致性来提升弱目标检出率。
3. **事件级测量评价**：建立事件检测率、虚警率、定位误差、类型检索准确率、工况鲁棒性和不确定性评估的统一测量体系。

## 4. 新的总实验框架

建议将整体流程改成下面这条主线：

1. 真实背景库构建
2. 仿真目标库构建
3. 半实物目标注入
4. 多视图信号增强
5. 候选事件生成
6. 模板库匹配与重打分
7. 事件级融合决策
8. 指标统计与泛化验证

可以用如下数学形式统一描述：

真实背景轨迹记为：

```math
x_{\mathrm{bg}}(t)
```

目标仿真响应记为：

```math
s(t; \theta)
```

其中 `\theta` 表示目标类别、深度、姿态、横向偏移、尺度、磁矩强度、传感器相对几何等参数。

将目标注入真实背景后得到半实物观测：

```math
x_{\mathrm{obs}}(t) = x_{\mathrm{bg}}(t) + \alpha \, s(t-\tau; \theta) + \varepsilon(t)
```

其中：

1. `\alpha` 控制目标强度或 SNR。
2. `\tau` 控制目标出现位置。
3. `\varepsilon(t)` 表示额外测量扰动、对齐误差或建模偏差。

对观测信号做多视图增强，得到多个增强通道：

```math
y_k(t) = F_k(x_{\mathrm{obs}}(t)), \quad k = 1, 2, \dots, K
```

再与模板库做匹配：

```math
M_k(\tau, \theta) = \operatorname{Match}(y_k, T_k(\cdot; \theta, \tau))
```

最终事件级分数写成：

```math
S(\tau) = \max_{\theta} \sum_{k=1}^{K} w_k M_k(\tau, \theta) - \lambda C(\tau, \theta)
```

其中 `C(\tau, \theta)` 是几何不一致性或多传感器不一致性惩罚项。

## 5. 数据层设计

### 5.1 真实背景库

新的主任务必须建立一个**真实背景库**，它不再只是原始输入，而是实验体系的底座。

建议做法：

1. 从现有实测航迹中整理出背景轨迹，保留 `sensor_a`、`sensor_b`、`mean_field`、`gradient_field`、`axis`、`sample_index` 等列。
2. 以“整条航迹文件”为基本划分单元做训练/验证/测试切分，避免同一条轨迹泄漏到不同集合。
3. 保留采样顺序，不要只按空间坐标重排，因为当前实现已经明确依赖 `sample_index` 保持真实采集顺序。
4. 为每条背景轨迹记录元数据，包括采样间隔、基线距离、航向、测量日期、传感器高度、环境干扰水平。

### 5.2 仿真目标库

你已有“船锚、锚钩、潜艇”等模型资源，这正是新的核心资产。

建议把它们从“辅助说明材料”升级为**目标仿真模板库**：

1. 目标类别：船锚、锚钩、潜艇、一般铁磁杂物。
2. 几何参数：尺寸、姿态角、埋深/离底高度、横向偏移、目标轴向朝向。
3. 测量参数：传感器间距、传感器高度、轨迹相对目标位置、采样步长。
4. 物理参数：等效磁矩、磁化方向、材料强度缩放因子。

建议每一种目标都离散出一组参数网格，形成模板族，而不是只生成一条理想曲线。

### 5.3 半实物注入策略

这是整篇文章能否成立的关键。

建议用“真实背景 + 参数化仿真目标注入”的方式构造标注样本：

1. 在真实背景轨迹的随机位置注入一个目标。
2. 控制注入强度，形成多档 SNR 或幅值比例。
3. 控制目标长度尺度，使其覆盖弱目标、可见目标和强目标。
4. 加入轻微轴向偏移、幅值失配、形状失配，模拟仿真与实测之间的域偏差。
5. 生成“无目标背景样本”作为负类。
6. 生成“硬负样本”，例如选取真实背景中自然起伏较强但未注入目标的片段。

建议至少覆盖以下工况：

1. `SNR / 强度`：弱、中、强三档。
2. `深度`：浅、中、深三档。
3. `姿态`：多个朝向。
4. `横向偏移`：轨迹正上方、偏移、远偏移。
5. `目标类别`：船锚、潜艇、其他铁磁体。

### 5.4 数据集划分原则

为了让结论足够强，不能只做随机切分。

建议同时做三类划分：

1. **背景独立划分**：不同背景轨迹不能交叉到训练与测试。
2. **参数独立划分**：测试集中使用未见过的深度、姿态或偏移组合。
3. **形态独立划分**：例如训练只用部分船锚外形，测试引入锚钩或潜艇。

这会让论文更像“泛化能力验证”，而不是“在合成数据上刷分”。

## 6. 方法层设计

### 6.1 重新定义“前端增强”的角色

当前仓库中的高通、EMD、VMD、低秩稀疏、OBF 等方法不需要删除，但要**降级为增强模块或基线模块**，不再是论文的最终目标。

新的定位应该是：

1. 它们负责增强目标响应。
2. 它们为后续模板匹配提供更干净的输入。
3. 它们本身不再被解释为“已经完成了目标检测”。

换句话说，分解和滤波只作为 `F_k(\cdot)` 的候选实现。

### 6.2 多视图表征

当前代码已经有三种表征雏形：

1. `single_channel`
2. `real_concat`
3. `dual_mode`

这部分非常适合保留，但要换一个说法：

1. `single_channel`：总场增强视图。
2. `real_concat`：总场 + 梯度联合视图。
3. `dual_mode`：共模/差模/耦合/显著性视图。

建议在新框架中将多视图表征统一为：

1. **幅值视图**：总场、残差总场。
2. **梯度视图**：梯度场、梯度残差。
3. **结构视图**：耦合响应、局部对称性、峰宽、零交叉邻域形状。
4. **能量视图**：局部能量、包络、归一化响应。

### 6.3 候选事件生成

当前 `SimpleDetector` 不应废弃，建议保留并明确其角色为：

**高召回候选生成器（proposal generator）**

它只负责：

1. 去背景
2. 局部稳健归一化
3. 低门限抓取候选片段

它不再承担最终分类决策。这样当前代码资产可以直接复用。

### 6.4 模板库匹配

这会成为新的核心算法层。

建议做成**粗到细两级匹配**：

#### 一级：快速粗匹配

针对每个候选片段，先用低成本特征做模板筛选：

1. 归一化峰值
2. 峰宽
3. 左右不对称度
4. 梯度峰对位置关系
5. 双传感器幅比与相位关系

用这些特征快速从大模板库中召回前 `K` 个候选模板。

#### 二级：精细匹配

再对召回模板做高精度匹配：

1. 归一化互相关
2. 滑动匹配滤波
3. 动态时间对齐或局部伸缩匹配
4. 多通道联合相似度
5. 几何一致性惩罚

最终给出：

1. 最佳匹配分数
2. 最佳模板参数
3. 最佳目标类别
4. 目标中心位置
5. 置信度和备选模板分数差

### 6.5 多视图融合决策

建议最终得分不要只靠一个通道，而是做多视图融合：

1. 总场匹配分数
2. 梯度匹配分数
3. 双模态耦合分数
4. 候选生成器的异常显著性分数
5. 几何一致性分数

一个简单但有说服力的融合公式可以写成：

```math
S_{\mathrm{final}} = w_1 S_{\mathrm{proposal}} + w_2 S_{\mathrm{mean}} + w_3 S_{\mathrm{grad}} + w_4 S_{\mathrm{dual}} - \lambda P_{\mathrm{inconsistency}}
```

其中 `P_{\mathrm{inconsistency}}` 用来惩罚模板与双传感器响应之间的不一致。

### 6.6 开集拒识

如果只做“船锚 vs 潜艇”的硬分类，很容易在真实背景中失稳。

建议加入一个开集拒识规则：

1. 最佳匹配分数过低则拒识。
2. 第一名和第二名模板分数差过小则判为不确定。
3. 只在“候选成立 + 模板匹配可靠 + 多视图一致”同时满足时输出目标类型。

这样更符合真实测量系统的工程逻辑。

## 7. 实验设计

### 7.1 主实验一：半实物异常目标检测

目标：

检验在真实背景上注入仿真目标后，系统能否正确发现目标事件。

比较对象：

1. 当前各类前端增强 + 固定阈值
2. 当前 `SimpleDetector`
3. 仅模板匹配，不做增强
4. 增强 + 模板匹配
5. 多视图增强 + 多视图模板匹配 + 几何一致性融合

### 7.2 主实验二：目标定位精度

目标：

评估检测结果与真实注入位置之间的偏差。

建议指标：

1. 峰值定位误差
2. 事件中心误差
3. 事件区间 IoU

### 7.3 主实验三：目标类型检索

目标：

评估最优匹配模板的类别是否与注入目标一致。

建议指标：

1. Top-1 类别准确率
2. Top-3 模板召回率
3. 类别混淆矩阵

### 7.4 主实验四：工况泛化能力

目标：

验证在未见过的深度、姿态、偏移和背景轨迹上，方法是否仍稳定。

建议设置：

1. 未见深度测试
2. 未见姿态测试
3. 未见背景航迹测试
4. 未见目标形态测试

### 7.5 主实验五：真实数据案例研究

目标：

在未注入的真实轨迹上输出高置信候选，并结合专家判读或现场知识做定性验证。

这一步很重要，因为 TIM 很看重“真实测量意义”，不能只停留在半实物合成实验。

## 8. 评价指标体系

建议统一用事件级指标，而不是只看逐点分数。

### 8.1 检测指标

1. `Pd`：检测概率
2. `FAR/km`：单位距离虚警率
3. `Precision / Recall / F1`
4. `ROC-AUC`
5. `PR-AUC`

### 8.2 定位指标

1. 峰值坐标误差
2. 中心位置误差
3. 事件边界 IoU

### 8.3 识别指标

1. Top-1 类别准确率
2. Top-3 模板召回率
3. 宏平均 F1

### 8.4 鲁棒性指标

1. 不同 SNR 下的检测率曲线
2. 不同深度下的性能曲线
3. 不同姿态下的性能曲线
4. 不同背景复杂度下的虚警率变化

### 8.5 测量可信度指标

为了更像 TIM，建议加入：

1. 置信度校准误差
2. 多次随机注入下的均值 ± 标准差
3. 轨迹级 bootstrap 置信区间

## 9. 消融实验建议

要让论文扎实，建议至少做以下消融：

1. 无仿真注入校准，只用阈值检出。
2. 只用总场，不用梯度场。
3. 只用单视图，不用多视图融合。
4. 不用候选生成器，直接全局模板匹配。
5. 不用几何一致性惩罚。
6. 不用开集拒识。
7. 稀疏模板库 vs 稠密模板库。
8. 无域偏差注入 vs 有域偏差注入。

## 13. 推荐的论文结构

如果按 TIM 稿件来组织，建议全文结构如下：

1. 引言：真实背景中弱磁异常目标检测难点，现有方法多偏前端分解，缺少物理一致的目标级评价体系。
2. 测量模型：双传感器磁测几何、目标仿真响应建模、背景与目标叠加关系。
3. 半实物数据集：真实背景库、仿真目标库、参数化注入和数据划分策略。
4. 方法：多视图增强、候选生成、模板匹配、几何一致性融合和开集拒识。
5. 实验：检测、定位、识别、泛化、消融和真实案例研究。
6. 讨论：仿真到实测域偏差、测量噪声、模型可解释性和部署可行性。
7. 结论：总结检测与测量价值。

## 15. 最小可执行落地路线

如果你要尽快推进，建议按以下三个阶段实施。

### 阶段 A：先把论文问题立起来

1. 固化真实背景库。
2. 整理船锚、潜艇仿真模板。
3. 做半实物注入器。
4. 生成第一版带标注 benchmark。

### 阶段 B：把主方法跑通

1. 保留当前 `SimpleDetector` 作为高召回候选器。
2. 保留现有增强方法作为前端池。
3. 新增模板库匹配和多视图融合。
4. 输出事件级检测和定位结果。

### 阶段 C：把 TIM 证据链补齐

1. 补工况泛化实验。
2. 补消融实验。
3. 补真实案例研究。
4. 补不确定性与测量可信度分析。

## 16. 最终建议

一句话总结这次方向调整：

**不要再把“分解得好不好”作为最终目标，而要把“在真实背景中能否可靠发现并解释仿真目标对应的异常事件”作为最终目标。**

在这个新框架里：

1. 分解、滤波、低秩稀疏、EMD、VMD 都还可以保留。
2. 但它们只负责“增强”和“提供视图”。
3. 真正的主线应该是“半实物目标注入 + 多视图模板匹配 + 事件级测量评价”。

如果按这个方向推进，论文故事、实验闭环、方法可解释性和 TIM 契合度都会比原来的“分解对比”路线强很多。
